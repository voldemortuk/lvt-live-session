Base must be your fork's main — not anshajk/main. If the PR targets the original repo, Actions runs there, fails, and you'll see nothing.
Read this before you click “Create”
Two things people get wrong
🎯 Base the PR on your fork
GitHub often defaults the base to the original repo. Switch it so the PR reads your-username/main ← your-username/exp1. Wrong base = it runs on Anshaj's repo and fails.
📁 Change a watched path
The OWASP review only fires when the change (new file or edit) sits under one of these paths:
▸ backend/src/**
▸ frontend/**
▸ contracts/openapi.yaml
✅
Get both right and the PR opens with checks running — Claude posts its security & OWASP review straight onto the conversation.
